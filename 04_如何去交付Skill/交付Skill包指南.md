# 交付 Skill 包指南

本节以与 OpenClaw 对话为主线，把自研 Skill 从「压缩包」一路装到可稳定调用的状态。你会依次完成：上传完整 Skill 压缩包并解压到正确的目录；重启 Gateway 后，核对是否加载、能否列举与说明用法；为脚本侧准备并管理 Python 虚拟环境；最后再做一轮基础功能测试。适用于把 Skill 部署在本机或通过 Gateway 暴露的 OpenClaw 使用场景。

---

## 目标

- 通过对话方式完成 Skill 包交付。
- 掌握解压目录、`~/.openclaw/skills` 放置规则与 Gateway 重启后的验证方式。
- 会用命令行与对话两种方式确认 Skill 是否生效，并做简单功能测试。

---

## 与 OpenClaw 按轮次对话

### 第 1 轮：直接上传 Skill 的完整压缩包

**你说：**

> 我待会上传一个Skill的压缩包，解压后放置在～/.openclaw/skills目录
>
> （直接追加上传压缩包，注意：压缩包内不要包含测试使用的 .venv 虚拟环境文件）

**要点：**

- 在支持上传的界面中，上传 **zip/tar 等完整压缩包**，不要只传单个文件。
- 压缩包内应自包含：`SKILL.md`、引用文档、可执行脚本、依赖声明等，便于后续解压到固定目录。
- 重启 Gateway：按当前安装方式需要重启**Gateway（一般会安装的时候自动重启）**。重启是为了重新加载 `skills` 目录。

### 第 2 轮：验证 Skill 是否已经就绪

**你说：**

> 当前可用的 Skill 有哪些？
>
> feishu-bill-bitable 这个Skill 如何使用？

**要点：**

- 验证方式（二选一或都做）：对话中让 OpenClaw 查询是否已加载该 Skill，或命令行过滤技能名：openclaw skills list | grep 技能名或关键词。

### 第 3 轮：为 Skill 配置脚本运行环境

**你说：**

> 帮我检查/配置这个Skill所需要的环境

**要点：** 安装虚拟环境成功。

### 第 4 轮：Skill 功能测试

**你说：**

> 帮我记录账单（上传账单图片）

**要点：**

- 覆盖 **正常路径**（至少一条完整链路）。
- 若有 **边界情况**（缺参数、错误输入），可追加一轮对话补充测试。
