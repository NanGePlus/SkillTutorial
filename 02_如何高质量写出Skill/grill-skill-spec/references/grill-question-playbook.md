# 拷问对话格式

拷问会话**开始时读取本文件**，按下列格式执行；五维顺序为：目的与范围 → 触发场景 → 领域知识 → 输出格式 → 既有模式。输出收尾用 skill-requirements-template.md。

## 单题输出格式（每轮必须遵守）

每一问**单独一条回复**，结构固定如下（勿合并多问、勿省略推荐方案）：

```markdown
## （{维序号}）{维名称} · 第 {题号} 问

**{一句清晰的决策型问题？}**

（可选：1 句上下文，如仓库检索结果或对用户已说内容的承接）

**推荐方案**（基于当前信息与仓库检索，可直接拍板）：

> {2–4 句结论：写清本题的决策点；若题目含「范围 + 不做」，在同一段或下一句点明**明确不做**}

同意请回复 **接受**（或「OK」「就按这个」）；若要改，直接说明要改的点即可。

（可选，仅当与推荐差异大、且用户可能难拍板时）
**其他方向**（≤2 条，每条一行，勿再列 A/B/C 长菜单）：
- {备选 1：与推荐的关键差异}
- {备选 2：…}
```

**允许的调整**（仍算遵守格式）：

- 某维用户首答已覆盖该题 → **跳过该题**，在「已记录」中写明依据，直接进入该维下一题。
- 需要展示模板时 → 在推荐方案下增加 markdown 代码块作为示例。
- 用户回复 **接受** / 同义确认 → 将推荐方案全文记入「已记录」，不重复追问。

## 用户回复后（维内）

在用户作答后、下一问之前，**必须先写「已记录」**（2–5 条 bullet 或一张小表），再出下一题：

```markdown
已记录：
- **{决策点}**：{用户选择或改写结论}
```

若用户回答含糊，**不进入下一题**，用同题号收窄并**重写一版更具体的推荐方案**（仍附「接受 / 说明修改」指引）；勿退回 A/B/C 选择题菜单。

## 维间过渡（每维最后一题答完后）

在进入下一维**第 1 问**之前，输出该维**收口摘要**（表格或 bullet），标题固定：

```markdown
（{维序号}）{维名称}已对齐，摘要如下：

| 项 | 共识 |
|---|---|
| … | … |
```

然后空一行，开始下一维的第 1 问（仍用「单题输出格式」）。

## 出题技巧

- 收紧模糊用语（二选一或给出定义）
- 用具体场景压测（正常 / 边界 / 失败）
- 指出前后矛盾与遗漏，要求用户当场拍板
- 推荐方案须**贴当前 Skill 主题**与已对齐前文撰写，禁止每题照搬无关示例；方案要具体到能写进规格摘要，避免空泛原则句
